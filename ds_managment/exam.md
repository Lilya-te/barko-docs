# Задание

## Бизнес-контекст
SaaS для B2C-сегмента — владельцев собак из приюта в первые 90 дней после пристройства в крупных городах РФ.

### Основная проблема
Уровень возврата собак в приют из-за несовпадения ожиданий новых хозяев с реальностью, вызванного недостаточной подготовкой, поддержкой и адаптацией в первые 90 дней составляет 45%.

Масштаб возвратов
- Ежегодно приюты пристраивают ~30 600 собак.
- 30% (9 180 собак) в среднем в год возвращаются обратно, создавая кризис доверия и повторную травму.
Финансовый удар по приютам
- Один возврат обходится минимум в 45 000 руб. (3 мес. содержания по 15 000 руб./мес.).
- Суммарные прямые годовые потери: ~620 млн руб. Эти средства не идут на спасение новых животных с улиц.
Системные последствия
- Возвраты блокируют места и ресурсы, не давая приютам снижать общую численность 4,1 млн бездомных животных.

**Вывод о востребованности**
Проблема сложной адаптации носит массовый характер, финансово ощутимый для приютов, эмоционально тяжёлый для владельцев, системный характер из-за отсутствия стандартизированной цифровой поддержки.
Результаты теоретического анализа и CustDev подтверждают гипотезу о необходимости:
- раннего прогнозирования адаптационных трудностей,
- персонализированного плана действий,
- структурированной системы сопровождения владельца.
Таким образом, существует подтверждённый спрос на цифровое решение, направленное на снижение возвратов собак в приюты в РФ."

## ML-решение
Проект представляет собой цифровой сервис для поддержки владельцев собак из приюта в период адаптации. Основой продукта является технология машинного обучения, которая позволяет прогнозировать паттерны поведения собаки в период адаптации, риск возврата в приют и тип возможных поведенческих проблем.

### Основная техническая идея и конструктивные особенности
Центральная идея: использование данных о поведении собак на ранних этапах адаптации для построения модели прогнозирования поведения и персонализированных рекомендаций для владельцев.

### Источники данных для обучения модели:
Одним из самых универсальных методов оценки поведения собаки является C-BARQ анкетирование. В открытых источниках можно найти множество накопленных данных о анкетировании владельцев собак этим методом. 
Планируется адаптация вопросов C-BARQ и C-BARQ-short (сокращенная версия) для построения моделей прогнозирования.
ML-модели будут решать следующие задачи: 
- прогнозирование рисков возникновения поведенческих проблем
- прогнозирование риска возврата
- прогнозирование паттернов поведения

**Примеры поведенческих проблем:**
- Сепарационная тревога
- Агрессия к другим животным
- Агрессия к людям
- Высокая возбудимость
- Различные виды страхов
- Пикарцизм
- Депрессия
*... список поведенческих проблем и паттернов поведения будет разработан совместно со специалистами по поведению собак*
Прогнозирование риска возврата будет возможно после сбора данных с помощью зооволонтеров 

### Каскад моделей машинного обучения:
- Модель прогнозирования рисков возникновения поведенческих проблем первого дня
- Модель прогнозирования рисков возникновения поведенческих проблем первой недели
- Модель прогнозирования рисков возникновения поведенческих проблем первого месяца
- Модель прогнозирования рисков возникновения поведенческих проблем второго месяца
- Модель прогнозирования рисков возникновения поведенческих проблем третьего месяца
- Модель прогнозирования паттернов поведения первого месяца
- Модель прогнозирования паттернов поведения первого года
- Модель прогнозирования риска возврата (сложная так как обучающие данные найти проблематично)

### Архитектура системы:
- Сбор данных: автоматизированный чат-бот подключается к владельцу собаки в первый день появления животного дома и собирает данные по контрольным вопросам.
- Хранение данных: все данные сохраняются в реляционной базе данных PostgreSQL.
- Разметка данных: специалисты по поведению собак (зоопсихологи, кинологи) предоставляют рекомендации и классифицируют типы поведенческих проявлений, что формирует целевые признаки для обучения модели.
- Обработка данных и обучение модели: при формировании датасета минимум из 200 кейсов запускается обучение моделей машинного обучения (дерево решений, случайный лес, градиентный бустинг, ансамбли).
- Метрики качества: для оценки пригодности модели на этапе MVP используется метрика ROC-AUC ≥ 0,75.
Используемые технологии и инновации:
- ML-технологии: дерево решений, случайный лес, градиентный бустинг, ансамбли; автоматизированная разметка данных с экспертной поддержкой.
- Backend: Python, Jupyter Notebook для разработки и обучения модели; REST API для интеграции с пользовательским интерфейсом.
- Базы данных: PostgreSQL для хранения структурированных данных, MongoDB для хранения логов и необработанных данных.
- API для обращения ко внешней LLM

### Инфраструктура: 
- облачные вычислительные сервисы для обучения моделей и хранения данных, масштабируемая архитектура для будущего расширения.
Интерфейс пользователя: 
- интерактивный план адаптации с уведомлениями и возможностью корректировки ответов

### Ключевые технические параметры продукта:
- Минимальный датасет: ≥ 200 кейсов адаптации собак.
- Целевые признаки: прогнозируемое поведение, риск возврата, тип поведенческих проблем.
- Метрика качества модели: ROC-AUC ≥ 0,75 на этапе MVP.
- Система контрольных точек: 1-й день, 2-й день, 1-я неделя, 2-я неделя, 3-я неделя, 1-й месяц, 2-й месяц, 3-й месяц (может изменяться в зависимости от рекомендаций модели и зоопсихолога).

## Риски и митигация

| Риск | Почему важен | Митигация |
|---|---|---|
| Недостаток данных для обучения модели риска возврата | Данных по возвратам собак мало, они разрознены и часто не размечены | Начать с модели прогнозирования поведенческих проблем на C-BARQ/C-BARQ-short, параллельно собирать собственный датасет через приюты и волонтеров |
| Низкое качество или неполнота анкет | Владельцы могут отвечать неточно, пропускать вопросы или субъективно оценивать поведение собаки | Использовать короткие анкеты, контрольные вопросы, подсказки с примерами поведения, повторные замеры в контрольных точках |
| Ошибочные рекомендации модели | Неверная рекомендация может ухудшить адаптацию собаки или снизить доверие к сервису | На MVP использовать модель как decision-support, а не как автоматическое экспертное заключение; валидировать рекомендации зоопсихологом |
| Низкое вовлечение владельцев | Если владельцы не проходят анкеты и не следуют плану, сервис не даст эффекта | Делать план коротким и практичным, добавить напоминания, прогресс, быстрые советы и поддержку в чат-боте |
| Недоверие приютов и волонтеров к цифровому решению | Без партнеров сложно получить данные и внедрить сервис в процесс пристройства | Запустить пилот с 1-2 приютами, показать снижение возвратов/обращений за помощью, дать приютам доступ к агрегированной аналитике |
| Ложноположительные прогнозы риска | Сервис может помечать собаку как «рискованную», хотя серьезных проблем не будет | Выбрать высокий Recall, но явно объяснять прогноз как вероятность, а не диагноз; давать мягкие профилактические рекомендации |
| Ложноотрицательные прогнозы риска | Сервис может пропустить серьезную проблему, и владелец останется без помощи | Отслеживать тревожные симптомы независимо от модели, добавлять правила-алерты и возможность быстро обратиться к специалисту |
| Юридические и этические риски | Рекомендации касаются поведения животных и решений владельца, возможна ответственность за вредные советы | Добавить дисклеймер, что сервис не заменяет ветеринара/кинолога; опасные случаи перенаправлять к специалисту |
| Рост стоимости экспертной разметки | Для качественных рекомендаций нужны зоопсихологи, это может ухудшить экономику | Размечать только сложные/спорные кейсы, использовать шаблоны рекомендаций, постепенно автоматизировать повторяющиеся сценарии |
| Смещение данных | Данные из крупных городов и активных пользователей могут плохо работать для других регионов/типов владельцев | Собирать данные из разных приютов и регионов, мониторить качество модели по сегментам, регулярно дообучать модель |

## Цикл проекта

1. Формализация бизнес-задачи и критериев успеха:
   - определить целевую бизнес-метрику: снижение доли возвратов собак в приют в первые 90 дней;
   - определить продуктовые метрики: completion rate анкет, NPS/удовлетворенность, retention;
   - определить ML-метрики: ROC-AUC, Recall, Precision для прогноза поведенческих рисков.

2. Проектирование данных и этического контура:
   - определить состав собираемых признаков: профиль собаки, данные приюта, ответы владельца, контрольные точки адаптации;
   - получить согласие пользователей на обработку данных;
   - обеспечить обезличивание данных и безопасное хранение в БД.

3. Сбор и разметка данных:
   - использовать открытые источники и адаптированные анкеты C-BARQ/C-BARQ-short;
   - организовать сбор данных через владельцев и волонтеров;
   - провести экспертную разметку кейсов зоопсихологами и кинологами.

4. Разработка baseline-решения:
   - реализовать простую версию без ML: правила, статьи, стандартные рекомендации;
   - зафиксировать baseline по продуктовым метрикам и качеству рекомендаций.

5. Обучение и валидация ML-модели:
   - обучить модели для прогноза поведенческих проблем и риска возврата;
   - разделить данные на train/validation/test;
   - сравнить алгоритмы: дерево решений, случайный лес, градиентный бустинг;
   - выбрать модель по ROC-AUC, Recall и экспертной интерпретируемости.

6. Экспертная проверка рекомендаций:
   - проверить рекомендации модели с участием зоопсихологов;
   - исключить вредные или неэтичные рекомендации;
   - настроить правила эскалации сложных случаев к специалисту.

7. Пилотное тестирование и A/B-тест:
   - запустить сервис на группе владельцев собак из приюта;
   - сравнить контрольную версию без ML и экспериментальную версию с ML-персонализацией;
   - оценить completion rate, NPS, частоту обращений за помощью и долю возвратов.

8. Релиз MVP:
   - развернуть backend, frontend, БД и ML-модель в облачной инфраструктуре;
   - подключить мониторинг технических и продуктовых метрик;
   - подготовить процесс поддержки пользователей.

9. Мониторинг в продакшене:
   - отслеживать качество модели, data drift/model drift, долю ложноположительных и ложноотрицательных прогнозов;
   - анализировать негативную обратную связь и случаи возврата собак;
   - контролировать стабильность сервиса и стоимость эксплуатации.

10. Итерационное улучшение:
   - регулярно пополнять датасет новыми кейсами;
   - дообучать модель по мере накопления данных;
   - добавлять модели для разных этапов адаптации: первый день, первая неделя, первый месяц, три месяца;
   - обновлять рекомендации на основе обратной связи и экспертной оценки.

## Свяжите ML-метрики с бизнес-метриками

Оценка эффективности решения по бизнес-метрикам: 
- уровень завершения анкет >12%
- удовлетворённость сервисом (NPS) будет не ниже 50

Оценка эффективности решения по ml-метрикам:
- ROC-AUC > 0,75
ROC-AUC Оценивает качество модели во всех возможных порогах классификации. Показывает соотношение истинно-положительных (TPR) и ложно-положительных (FPR) результатов. При корректных предсказаниях модели, пользователь будет обращаться к ней на каждом этапе адаптации собаки, что вляиет на завершение прохождения всех анкет.

- Recall > 0,95
Recall Доля правильно определенных положительных объектов от общего числа реальных положительных объектов. Критична, когда цена ошибки ложно-отрицательного ответа высока. В нашем случае выбираем эту метрику, так как лучше предупредить владельца собаке о риске возникновения проблемы и рассказать, что делать в случае возникновения, чем пропустить риск и оставить владельца наедине с проблемой. Влияет на удовлетворенность пользователей.


## План A/B-теста для валидации решения
Цель теста — проверить, дает ли ML-персонализация адаптационного плана измеримый прирост по сравнению с базовым цифровым сопровождением без ML.

Гипотеза: владельцы собак из приюта, которые получают персонализированный план адаптации на основе ML-прогноза рисков и регулярных контрольных точек, чаще доходят до конца 90-дневной программы и выше оценивают полезность сервиса, чем владельцы, получающие только статьи, базовое анкетирование и общие рекомендации.

Дизайн теста:
- С помощью волонтеров и партнерских приютов привлечь владельцев, которые недавно взяли собаку из приюта.
- Участников случайным образом разделить на две группы: контрольную и экспериментальную.
- Контрольная группа получает сервис без ML: статьи, базовое анкетирование, стандартные рекомендации и напоминания.
- Экспериментальная группа получает сервис с ML: прогноз риска поведенческих проблем, персонализированный план адаптации, корректировку рекомендаций по контрольным точкам.
- Срок наблюдения — 90 дней, так как именно первые 3 месяца являются критичным периодом адаптации.

Выборка: для пилота достаточно 100-120 владельцев собак, по 50-60 человек в каждой группе. Если в пилоте эффект будет заметен, тест масштабируется до 300+ участников для более надежной статистической проверки.

Основная метрика успеха: доля пользователей, дошедших до конца 90-дневной программы адаптации и заполнивших ключевые анкеты по контрольным точкам.

Дополнительные метрики:
- удовлетворенность сервисом / NPS;
- доля владельцев, которые хотя бы раз воспользовались рекомендацией;
- число обращений за помощью по поведенческим проблемам;
- доля возвратов собак в приют в течение 90 дней;
- среднее количество заполненных анкет на одного пользователя.

Критерий успеха: экспериментальная группа показывает статистически значимый рост дохождения до конца программы и NPS по сравнению с контрольной группой. Для пилота целевой эффект: рост завершения программы минимум на 15-20% и NPS не ниже 50.

Критерий остановки: тест завершается через 90 дней после подключения последнего участника либо раньше, если набрана минимальная выборка и экспериментальная группа показывает устойчивое ухудшение ключевых метрик или рост негативной обратной связи. После завершения сравниваются контрольная и экспериментальная группы, фиксируется вывод о целесообразности внедрения ML-персонализации.

## Примеры использования продукта (Use Case)
- Волонтер или сотрудник приюта заполняет стандартизированную анкету с характеристиками собаки (пол, возраст, вес, порода/метис, опыт проживания, поведенческие особенности).
- Владельцу при активации приложения предлагается подтвердить и дополнить данные или внести, если данных нет.
- Данные автоматически сохраняются в базе.
- Формирование плана адаптации: система анализирует ответы и выдает индивидуальный план на 1 неделю с прогнозом поведения и рекомендациями специалиста.
- Контрольные точки: через установленные интервалы владелец получает напоминания, заполняет обновлённые данные, а система корректирует план на следующий период.
- Прогнозирование риска: владелец и приют получают прогноз риска возврата собаки, что позволяет предпринять профилактические меры.
- Поддержка в чат-боте, ответ на возникающие вопросы

## Конкурентные преимущества
- Структурированность данных и персонализация:  объединяет сбор данных, экспертную разметку и прогнозирование поведения в одном сервисе.
- Экспертная поддержка: рекомендации формируются с участием зоопсихологов, что обеспечивает высокую достоверность.
- Метрика качества: использование ROC-AUC ≥0,75 гарантирует приемлемую точность прогноза на MVP этапе, что делает сервис надежным инструментом для владельцев и приютов.
