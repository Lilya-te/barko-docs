# Adoption Return (возврат в приют)

Кандидат на **операциональное определение** неудачной адаптации для K3.

## Определение

Собака считается с **неудачной адаптацией**, если после пристройства (`home_started_at`) она **вернулась в приют** (`back_shelter_at` не пуст).

```text
target = 1  if back_shelter_at is not null  (после последнего пристройства)
target = 0  if пристроена и не вернулась
```

## Обоснование

- Прямой продуктовый outcome: владелец не смог удержать собаку.
- Измерим на реальных данных HelpDog — см. [helpdog-adoption-eda](../ml/helpdog-adoption-eda.md).
- ~57% возвратов среди 46 пристроенных собак — высокая базовая частота.

## Ограничения

| Ограничение | Комментарий |
|-------------|-------------|
| Малый объём | 46 пристроенных, 26 возвратов — недостаточно для train |
| Нет C-BARQ | HelpDog-данные не содержат поведенческих шкал |
| Неполная история | Не все возвраты могут быть зафиксированы на форуме |
| Причины возврата | В тексте `возврат` часто «причина неизвестна» |
| Альтернативные таргеты | Сложная адаптация без возврата; оценка эксперта; C-BARQ-пороги |

## Связь с K3

- **HelpDog:** label source для валидации гипотез (продуктовые данные).
- **C-BARQ датасеты:** train baseline без прямого label «возврат» — нужен proxy (поведенческие риски) или external label engineering.

См. [k3-risk-prediction](../components/k3-risk-prediction.md).
