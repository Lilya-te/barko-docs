# HelpDog Forum Adoptions

Внутренний продуктовый датасет: история пристройств собак с форума HelpDog. Источник для EDA и потенциальный label source для K3.

## Файлы

| Файл | Строк | Описание |
|------|------:|----------|
| `analytics/dogs.csv` | 90 | Список собак на форуме |
| `analytics/adoptions.csv` | 125 | События пристройства и возврата |

## Схема `dogs.csv`

| Поле | Описание |
|------|----------|
| `name` | Имя собаки |
| `link` | Ссылка на тред форума |
| `поступление` | Текст о поступлении (дата, передержка) |
| `id` | ID треда / собаки |
| `последнее_сообщение` | Ссылка на последнее сообщение |

## Схема `adoptions.csv`

| Поле | Описание |
|------|----------|
| `dog_id` | FK → `dogs.id` |
| `shelter_started_at` | Дата начала передержки/приюта |
| `home_started_at` | Дата пристройства (пусто = не пристроена) |
| `back_shelter_at` | Дата возврата в приют (пусто = не вернулась) |
| `пристройство` | Текст сообщения о пристройстве |
| `возврат` | Текст сообщения о возврате |

## Статистика (из EDA)

| Метрика | Значение |
|---------|----------|
| Собак на форуме | 90 |
| Пристроены ≥1 раз | 46 |
| Вернулись в приют ≥1 раз | 26 |
| Доля возвратов среди пристроенных | ~57% |

См. [helpdog-adoption-eda](../ml/helpdog-adoption-eda.md).

## Релевантность для K3

- **Потенциальный таргет:** бинарный `returned_to_shelter` после пристройства — см. [adoption-return](../concepts/adoption-return.md).
- **Ограничение:** нет C-BARQ-признаков; 90 собак — слишком мало для самостоятельного ML baseline.
- **Роль:** валидация гипотез о «неудачной адаптации» на реальных данных; будущая связка с анкетой Barko.

## Очистка данных (логика ноутбука)

Для собак с несколькими циклами пристройства:
- удалить промежуточные пристройства без возврата;
- оставить последнее пристройство (если без возврата) или записи с `back_shelter_at`.
