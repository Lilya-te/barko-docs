# HelpDog Adoption EDA

Результаты exploratory analysis из `analytics/helpdog.ipynb`.

## Цель анализа

Оценить масштаб и частоту **возвратов в приют** после пристройства — как прокси неудачной адаптации в реальных данных HelpDog.

## Данные

- `dogs.csv` — 90 собак
- `adoptions.csv` — 125 записей о событиях пристройства/возврата

См. [helpdog-forum-adoptions](../datasets/helpdog-forum-adoptions.md).

## Метод

1. Отфильтровать записи с непустым `home_started_at` (собака была пристроена).
2. Для собак с несколькими пристройствами — очистить историю (см. логику в ноутбуке).
3. Посчитать уникальных собак с непустым `back_shelter_at`.

## Результаты

| Метрика | Значение |
|---------|----------|
| Собак на форуме | 90 |
| Пристроены ≥1 раз | 46 (51% от всех) |
| Вернулись в приют ≥1 раз | 26 |
| **Доля возвратов** (от пристроенных) | **~57%** (26/46) |

## Интерпретация для Barko

- Возврат в приют — сильный **продуктовый сигнал** неудачной адаптации.
- 57% — высокая базовая частота; при дисбалансе классов для K3 важны `PR-AUC` и `Recall` high-risk.
- Выборка мала (46 пристроенных) — статистика ориентировочная, не для финального benchmark.

## Что ноутбук НЕ делает

- Не обучает модели (`RandomForest`, `LogisticRegression` и др.).
- Не считает ROC-AUC, PR-AUC, Brier.
- Не связывает собак с C-BARQ-признаками.

Таблица моделей K3 в [benchmarks-and-metrics](benchmarks-and-metrics.md) по-прежнему **пустая** — baseline на C-BARQ ещё не запущен.
